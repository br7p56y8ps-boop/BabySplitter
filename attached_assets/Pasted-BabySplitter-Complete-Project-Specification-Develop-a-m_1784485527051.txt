Tech Stack & Deployment (Free Tier)
Code hosting: GitHub (private or public repo)
Frontend framework: React (Next.js recommended for PWA support, routing, and easy Vercel deploys)
Frontend hosting: Vercel — auto-deploys from GitHub on every push, free SSL, free tier sufficient for small user groups
Backend / Database / Real-time sync: Supabase (Postgres) — use Supabase real-time subscriptions for live sync across Expenses, Settlements, History, Chat, and Notifications
Auth: None required (per Member Selection spec) — use local identity storage only, no Supabase Auth needed
File/Image storage: Supabase Storage (free tier) for app icon and any uploaded assets
PWA requirements: Web Manifest, service worker for offline caching, installable icons/splash screens generated via a tool like pwa-asset-generator
Offline behavior: If an action (add/edit expense, chat message, settlement) is performed with no network, queue it locally and auto-sync once reconnected. Show a “pending sync” indicator on queued items until confirmed synced.
Push notifications: In addition to the in-app notification bell, implement real push notifications via the service worker (e.g. Web Push API) so members are notified even when the app isn’t open.
All services above remain free at this app’s expected scale (fixed small set of members, low read/write volume).
App is intended for a closed group of known, trusted friends — no authentication or per-user access control is required.

UI Design Requirements
The app must look and feel like a modern, current-generation app — not a generic template or dated/basic UI. Avoid plain flat colors, boxy default
components, or anything that looks like a default Bootstrap/Material starter theme. Every screen should feel premium, deliberate, and visually polished, on par with top-tier consumer apps released in 2025–2026.
Implement true iOS Liquid Glass aesthetics with translucent frosted glass surfaces, subtle blur, soft highlights, realistic reflections, and depth.
Convert all cards, navigation bars, tabs, sidebars, modals, dialogs, dropdowns, buttons, input fields, floating panels, and widgets into glassmorphic components.
Use rounded corners (16–28px), soft shadows, thin glass borders, and layered transparency.
Replace all icons with glass-styled icons featuring subtle depth and lighting while maintaining consistency throughout the UI.
Add smooth micro-interactions, hover effects, press animations, spring transitions, and fluid page transitions inspired by iOS.
Support both Light Mode and Dark Mode (black theme) with seamless switching and proper contrast for accessibility.
Ensure typography remains clean and readable using Apple’s visual hierarchy with generous spacing.
Use vibrant accent colors only where appropriate, allowing glass surfaces to remain the primary visual element.
Maintain responsive behavior across desktop, tablet, and mobile devices.
Optimize for performance by minimizing expensive blur effects and ensuring smooth animations (60fps where possible).
Preserve accessibility (WCAG), keyboard navigation, and touch interactions.
Do not alter the site’s structure, features, or business logic—only modernize the visual design.
Goal: Create a polished, premium, futuristic interface that feels like a native iOS experience with Liquid Glass-inspired glassmorphism throughout the entire website, including every card, tab, icon, navigation element, and interactive component.

After saving, display the expense under Recent Expenses. Every expense appears as an expandable card.
Collapsed Card: Expense Title, Total Expense, Date
Expanded Card: Paid By, Participants, Edit Expense, Delete Expense, Reset Expense
All fields remain editable after creation. If an expense has already been partially settled, already-settled transactions must remain frozen (unaffected by the edit); only the unsettled remainder is recalculated against the new values.
Temporary (per-expense) members are treated identically to preset members within that expense’s settlement calculations, “who owes whom” breakdown, and History records — they simply have no Settings profile of their own.
Settlement
Display every expense as an expandable card.
Top-right: Settle All
Each expense supports: Settle this expense only. Select multiple expenses. Settle remaining expenses (settles every unsettled transaction across all expenses not yet individually settled). Settle All settles every outstanding transaction across all expenses, including any already selected for individual settlement — it does not conflict with or duplicate prior individual settlements, since settled transactions are always excluded from further calculation. Expanded card displays: Expense Title, Total Amount, Paid By, Participants, Equal share, Who owes whom Example:
A	→ B : ₹250     [Settle]
B	→ C : ₹180     [Settle]
Individual settlement rules:
Only that debtor-creditor transaction is settled
Expense becomes Partially Settled
Settled transaction must never be counted again
Previously settled transfers remain visible
